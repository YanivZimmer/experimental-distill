# Expected Data Files

## Overview

The distillation pipeline expects specific data files in the `data/` folder. Here's what you need:

## 📥 Required Input Files

### 1. `data/langfuse_test.json` (2.4 MB)

**Description:** Raw security alerts from Langfuse
**Format:** Array of alert objects

```json
[
  {
    "id": "af24e5a7-ac15-5009-967d-f5c4f6ef0a59",
    "input": {
      "raw_alert": {
        "name": "Alert Name",
        "type": "Detection",
        "device": {
          "hostname": "example-host",
          "device_id": "..."
        },
        "file_name": "suspicious.exe",
        "sha256": "hash...",
        ...
      }
    }
  },
  ...
]
```

**Key fields:**
- `id`: Unique identifier (used to join with benchmark data)
- `input`: The raw alert data that will be sent to the model

### 2. `data/teacher_output.json` (46 KB)

**Description:** Teacher model (Gemini) outputs with ground truth labels
**Format:** Object with metadata and items array

```json
{
  "metadata": {
    "model": "gemini-2.0-flash-001",
    "test_items": 143,
    "items_processed": 8,
    ...
  },
  "metrics": { ... },
  "items": [
    {
      "item_id": "af24e5a7-ac15-5009-967d-f5c4f6ef0a59",
      "expected_label": "True Positive - Benign",
      "predicted": "malicious",
      "hit": 0,  // 1 = agreed with ground truth, 0 = disagreed
      "output": "{...}",  // Full structured JSON output
      "classification": {
        "primary_assessment": "High-Confidence Suspicious",
        "final_decision": "Escalate for Review",
        "severity": "Low",
        "justification": "The combination of..."
      },
      ...
    },
    ...
  ]
}
```

**Key fields:**
- `item_id`: Matches `id` in langfuse_test.json
- `expected_label`: Ground truth classification
- `hit`: 1 if Gemini agreed with ground truth, 0 if not
- `classification.justification`: Reasoning from teacher model
- `classification.final_decision`: Teacher's decision

## 📤 Generated Files

### 3. `data/train_distill.json` (103 KB)

**Generated by:** `scripts/prepare_dataset.py`
**Description:** Combined training data (alerts + teacher reasoning)

```json
[
  {
    "alert": { ... },  // Raw alert from langfuse
    "reasoning": "The combination of...",  // Teacher reasoning
    "classification": "Escalate for Review (Assessment: High-Confidence Suspicious, Severity: Low)",
    "metadata": {
      "item_id": "...",
      "expected_label": "...",
      "teacher_model": "gemini-2.0-flash-001"
    }
  },
  ...
]
```

**Filtering:**
- By default, only includes samples where `hit == 1` (Gemini agreed with ground truth)
- Use `--no-filter` flag to include all samples

### 4. `data/splits/` (Generated by split_dataset.py)

```
data/splits/
├── train.json       # 70% of data
├── val.json         # 15% of data
├── test.json        # 15% of data
└── split_metadata.json
```

Each file has the same format as `train_distill.json`, just split into train/val/test sets.

## 🔄 Data Flow

```
1. Raw Data
   ├── data/langfuse_test.json (alerts)
   └── data/baseline_benchmark_flash_gepa_v1.json (teacher outputs + labels)
          ↓
2. Prepare Dataset (scripts/prepare_dataset.py)
   → Joins alerts with teacher outputs
   → Filters to keep only Gemini agreements (hit == 1)
   → Outputs: data/train_distill.json
          ↓
3. Split Dataset (scripts/split_dataset.py)
   → Splits into train/val/test (70/15/15)
   → Prevents data leakage (similar alerts in same split)
   → Outputs: data/splits/{train,val,test}.json
          ↓
4. Training (training/train.py)
   → Loads from data/splits/
   → Trains model
   → Outputs: outputs/distilled_model/
```

## ❓ FAQ

**Q: Where do I get langfuse_test.json and baseline_benchmark_flash_gepa_v1.json?**
A: These should be provided or generated from your Langfuse system and Gemini benchmark runs.

**Q: Can I use different file names?**
A: Yes! Use command-line arguments:
```bash
python scripts/prepare_dataset.py \
  --langfuse data/my_alerts.json \
  --benchmark data/my_benchmark.json \
  --output data/my_training_data.json
```

**Q: What if my data has a different format?**
A: You'll need to modify `scripts/prepare_dataset.py` to match your data schema. The key requirements are:
1. Alerts with unique IDs
2. Teacher outputs with reasoning and classifications
3. Ground truth labels

**Q: How many samples do I need?**
A: Current dataset has 143 total items, ~4-8 after filtering. For production, aim for 100+ filtered samples.
